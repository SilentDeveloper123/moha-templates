MOHA Template — Full Documentation

Project Overview

MOHA Templates is a production-ready HTML/CSS template system for the Ministry of Home Affairs, Sri Lanka. It provides a complete set of reusable page templates, UI components, and a SCSS-based styling architecture built on top of Tailwind CSS (via CDN) and custom SCSS partials compiled with the Dart Sass compiler.

The project includes 50+ HTML pages covering all major sections of a government ministry website: home, divisions, institutions, services, news, governance, circuit bungalow booking, publications, circulars, downloads, and more.

Version: 1.0.0  |  License: Internal Use  |  Dependencies: Tailwind CSS, Font Awesome 6, jQuery 3.7, GLightbox, Sass

Quick Start / Setup

The project uses Dart Sass to compile SCSS to CSS. Node.js and npm are required.

1. Install Dependencies

npm install

2. Build CSS (production)

npm run build:css

Compiles SCSS to css/style.css with compressed output and no source map.

3. Watch for Changes (development)

npm run watch:css

Watches scss/ for changes and auto-recompiles.

4. Dev Mode with Source Maps

npm run dev:css

Compiles with expanded output and embedded source maps for debugging.

5. Open in Browser

Open index.html directly in a browser, or serve via any HTTP server (Live Server, VS Code's live preview, etc.).

Note: Tailwind CSS and Font Awesome are loaded via CDN — no build step required for them. jQuery and GLightbox are also loaded from CDN in each page's <head>.

SCSS Architecture

The SCSS follows a 7-1-ish partial architecture. All partials are imported into style.scss in a specific order for proper cascade.

Partial Breakdown

_variables.scssSCSS variables: $navy: #0A1931, $green: #43c000, $steel: #2D5F8A, $maroon: #801200 plus light opacity variants.
_base.scssExports SCSS vars as CSS custom properties (--navy, --green, etc.) on :root. Sets html { scroll-behavior: smooth } and base body font.
_loader.scssFull-screen navy overlay with a spinning green border circle and pulsing text. Hidden via .hidden class (opacity/visibility transition).
_topbar.scssSingle rule: navy background on .topbar with bottom border.
_dropdowns.scssAbsolute-positioned dropdowns with smooth fade+slide animation. Hover/focus-within triggers. Nested > .dropdown-menu pattern. Z-index layers: topbar (300), header (200).
_hero.scssFull-width hero with stacked background images (.slide-bg), gradient overlay, SVG pattern noise via pseudo-element, fade-in keyframe for slides (heroFade).
_ticker.scssHorizontal continuous scrolling news ticker. Uses inline-flex + translateX with will-change for performance.
_banner.scssInterior page banner with navy gradient background, optional image overlay, and SVG pattern noise.
_components.scssLargest partial — card hover effects (lift + shadow), card top-line accent, mobile nav (blur backdrop, slide-in links), accordion rotation, secondary menu slide, back-to-top fixed button, pagination active state, stat numbers in green, sub-navigation active border, custom table styles, scrollbar styling, sidebar link hover, footer social icon animations.
_lightbox.scssGLightbox overrides: rounded images, navy overlay with blur, custom close/prev/next button styling, gallery trigger hover overlay with search icon.
_responsive.scssThree breakpoint blocks: ≤639px (mobile), 640-1023px (tablet), ≥1024px (desktop). Adjusts hero font sizes, stat padding, gallery aspect ratios, back-to-top position, CTA buttons, and special tile layouts.

JavaScript Features

All JavaScript lives in js/main.js (183 lines) wrapped in jQuery's $(document).ready(). The file is loaded with defer in the page <head>.

Module Breakdown

Page LoaderHides #pageLoader on window load with a 3-second fallback timeout. Fades via CSS transition.
Mobile MenuHamburger toggles .nav-mobile with fadeIn. Close via close button or any nav link. Uses display: flex for proper layout.
Hero SliderCycles through .hero-slide and .slide-bg elements. Auto-starts at 5.5s interval. Click handlers for prev/next arrows and dots. Resets interval on manual interaction.
TabsClick .tab to switch. Removes active state from siblings, activates clicked tab, shows corresponding .tab-panel by data attribute ID.
Anchor ScrollSmooth-scrolls to #hash targets with 100px offset and 500ms animation.
Back to TopDynamically creates .back-to-top button. Shows after 500px scroll. Smooth scrolls to top on click.
Counter AnimationAnimates .stat-counter numbers from 0 to target in ~40 steps when .stats-bar enters viewport (IntersectionObserver).
News TickerContinuous horizontal scroll via requestAnimationFrame. Clones items for seamless loop. Nav buttons snap by one item width. Speed: 0.35px/frame.
AccordionClick .accordion-toggle to slideToggle .accordion-content. Toggles .open class for chevron rotation.
Secondary MenuSlide-in panel from left. Opens on click, closes via close button, link click, or click outside. Animated with CSS transitions.
LightboxInitializes GLightbox on .glightbox selector with touch, loop, zoom, and drag options.
Scroll AnimationsIntersectionObserver watches .scroll-animate elements. Applies .animate-fadeIn class with opacity 0→1 and translateY 30px→0. Falls back to inline style injection.

Tailwind CSS Configuration

Tailwind is loaded via CDN with a custom config that extends the default color palette with four brand colors:

<script>
tailwind.config = {
  theme: {
    extend: {
      colors: {
        navy: '#0A1931',
        green: '#43c000',
        steel: '#2D5F8A',
        maroon: '#801200'
      }
    }
  }
}
</script>

This allows usage of utility classes like:

bg-navyNavy background
text-goldGreen text
border-steelSteel border
from-navy to-steelGradient
bg-gold/1010% green opacity
shadow-green/25Green-tinted shadow

The rest of Tailwind's default utility set is available as usual (flex, grid, spacing, typography, shadows, etc.).

Responsive Breakpoints

The project uses three tiers from Tailwind's default breakpoint system, plus custom overrides in _responsive.scss.

NameMin WidthTailwind PrefixNotes
Mobile0(default)Single-column layouts, stacked navigation, compact spacing. Topbar hides contact hours. Hero at smaller font sizes.
Tablet640px (sm)sm:Multi-column grids activate. Topbar shows full contact info. Gallery maintains aspect ratios.
Desktop1024px (lg)lg:Full horizontal nav visible. Hero full size. Back-to-top shifts position. Sidebar nav appears (on docs).

Custom responsive rules in _responsive.scss handle:

  • ≤639px: hero font clamp, stat padding, gallery ratios, back-to-top reposition, CTA button sizing, mobile special tiles
  • 640-1023px: hero headline sizing, stat/ section padding adjustments, tablet gallery
  • ≥1024px: desktop gallery height fix

Additional responsive behavior is handled by Tailwind's responsive prefixes (sm:, lg:) directly in HTML classes throughout all templates.

How to Create a New Page

Follow these steps to create a new page that matches the existing templates:

  1. Copy template.html — This is the base skeleton with all required includes (Tailwind, Font Awesome, jQuery, GLightbox, css/style.css, js/main.js).
  2. Update the <title> — Set a descriptive title like Page Name — Ministry of Home Affairs, Sri Lanka.
  3. Add a <meta name="description"> — For SEO and accessibility.
  4. Keep the page loader — The <div class="page-loader"> must remain as the first body element.
  5. Add the Topbar — Copy from template.html or use the Topbar component below.
  6. Add the Header/Nav — Copy the <header> block with main navigation and hamburger.
  7. Add a Page Banner — Use the Page Banner component below, update the title and breadcrumb.
  8. Add page content — Wrap content in <div class="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-16">...</div>.
  9. Add the Footer — Copy from template.html or use the Footer component below.
  10. Add the Back-to-Top button — The JS creates it dynamically; no HTML needed (but can add manually as shown in the component).
  11. Test responsiveness — Check at mobile (320px), tablet (768px), and desktop (1440px) widths.
Tip: Use the Component Library section below to copy-paste ready-made sections. Each component shows a live preview and has a Copy button for its HTML code.

Customization Guide

The template is designed to be easily customizable. Here's how to change the look and feel.

Colors

Edit scss/_variables.scss to change brand colors, then re-run npm run build:css. Also update the tailwind.config in each page's <head> to match.

// _variables.scss
$navy: #0A1931;    // Dark blue — primary backgrounds, headings
$green: #43c000;   // Green — accent, buttons, links, highlights
$steel: #2D5F8A;   // Steel blue — secondary, gradients
$maroon: #801200;  // Maroon — warning/alert accent

Fonts

The base font stack is 'Inter', 'Segoe UI', system-ui, sans-serif. To change it:

  1. Replace the font in scss/_base.scss (body font-family) and in scss/_lightbox.scss (GLightbox title/desc).
  2. Add a new Google Fonts <link> in the <head> of each page (or update template.html if it's the base).

Spacing & Layout

The template uses Tailwind's default spacing scale. To adjust global padding/section spacing, modify _responsive.scss .section-pad and .card-pad values. For per-component spacing, adjust the Tailwind classes directly in HTML.

Border Radius

Standard radius values: 8px (inputs), 10px (doc cards), 12px (component cards, buttons), 50% (icons, avatars). Adjust in component SCSS or via rounded-* classes.

Shadows

Card hover uses 0 12px 40px rgba(0,0,0,0.08). Buttons use shadow-lg (Tailwind) with optional green tint: shadow-green/25. Dropdown menus use 0 20px 60px rgba(0,0,0,0.15).

Icons

All icons use Font Awesome 6 (free). Change icon by replacing the fa-* class. See Font Awesome icon catalog.

Deployment Notes

Deploying the MOHA template to a production server.

Prerequisites

  • Any static web server (Apache, Nginx, IIS, Netlify, Vercel, GitHub Pages)
  • No server-side processing required — all pages are static HTML

Build for Production

npm install
npm run build:css

This produces a minified css/style.css. The node_modules/ directory is not needed at runtime.

Deploy Steps

  1. Run npm run build:css to compile production CSS.
  2. Upload the entire moha-templates/ directory to your web server's document root.
  3. Ensure the server is configured to serve index.html as the default document.
  4. Test all pages and verify CDN resources (Tailwind, Font Awesome, jQuery, GLightbox) load correctly.

Performance Considerations

  • Optimize images before deployment — use WebP format where possible.
  • Replace placeholder images (placehold.co) with actual images.
  • Consider inlining critical CSS or using a proper build tool for production.
  • Enable gzip/brotli compression on your web server for HTML, CSS, and JS.
  • Set appropriate cache headers for static assets.

Offline / Intranet Deployment

For intranet environments without internet access, download the CDN resources and serve them locally:

  1. Download the Tailwind CSS script and host it at js/tailwind.min.js.
  2. Download Font Awesome and host at css/all.min.css (with the webfonts/ folder).
  3. Download jQuery and GLightbox, host locally.
  4. Update all <link> and <script> tags to point to local paths.
